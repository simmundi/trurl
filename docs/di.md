# Dependency Injection

Trurl is built on top of **Bento**, a powerful and lightweight Dependency Injection container.

Bento's core philosophy is that objects are created via a `Bento` instance, which is itself a Unit of Work. Instead of using reflection to find dependencies, Bento prefers to use **Factories** (often generated by an annotation processor).

## The Bento approach

In Trurl, you'll see `@WithFactory` on constructors. This informs the Bento annotation processor to generate a factory class (e.g. `EngineBuilderFactory`).

```java
public class MyService {
    private final Engine engine;

    @WithFactory
    public MyService(Engine engine) {
        this.engine = engine;
    }
}
```

## Configuring Trurl via Bento

Bento supports named parameters, which Trurl uses for configuration. You can register values in a Bento instance and then use them when building your Engine:

```java
Bento bento = Bento.createRoot();
bento.register(EngineBuilder.INITIAL_CAPACITY, 1_000_000);
Engine engine = bento.get(EngineBuilderFactory.IT).getEngine();
```

The `EngineBuilder` class defines several configuration constants:
- `trurl.engine.initial-capacity`
- `trurl.engine.capacity-headroom`
- `trurl.engine.session-cache-size`

Additional keys:
- `trurl.engine.attributeFactory`: to switch from the default `basic` (array-based) implementation.

For more details on Bento, please refer to its official documentation.
